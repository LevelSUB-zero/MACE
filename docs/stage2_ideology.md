# STAGE-2 IDEOLOGY & BOUNDARY (AUTHORITATIVE)

> **The Core Idea:**
> Stage-2 teaches MACE how to judge the past — not how to change the future.

---

## 0. Why Stage-2 Exists (Non-Negotiable Framing)

Stage-2 exists to answer one question only:
> “What would be worth remembering, if remembering were allowed?”

It does **not** answer:
- What should be remembered
- What should be acted on
- What should be trusted
- What should be executed

Stage-2 **never** changes system behavior.
It changes only records about past behavior.

This is the epistemic layer of MACE.
If Stage-2 were deleted tomorrow, Stage-1 behavior **must remain identical**.
That is the litmus test.

---

## A. STAGE-2 IDEOLOGICAL BOUNDARY (FOUNDATIONAL, HARD LAW)

### A.1 Definition of Intelligence Escalation
An intelligence escalation occurs if any learned artifact influences:
1. Execution order
2. Routing choice
3. Memory permanence
4. Agent selection
5. Threshold values
6. Future decision structure

Stage-2 **must not** create escalation pressure.

### A.2 Epistemic vs Agentic (Formal Distinction)

| Aspect | Epistemic | Agentic |
| :--- | :--- | :--- |
| **Time direction** | Backward-looking | Forward-looking |
| **Power** | Observational | Causal |
| **Output** | Judgments | Actions |
| **Authority** | Descriptive | Prescriptive |

Stage-2 is **epistemic only**.

### A.3 Explicit Prohibitions (Hard Fail)
These are fatal violations:
- MEM-SNN output read by Router
- Candidate score used as threshold
- Council label auto-triggering SEM write
- Learning influencing retries
- Adaptive scoring weights
- Feedback loops without governance

Each violation must:
1. Halt Stage-2
2. Trigger kill-switch
3. Mark logs as tainted
4. Require human intervention

## B. SHADOW MODE PRINCIPLE (PRECISE, ENFORCEABLE)

### B.1 Definition of Shadow Mode
Shadow mode means:
> A subsystem may predict, score, or rank, but its outputs are **observed only, never consumed**.

### B.2 Allowed Shadow Outputs
MEM-SNN may produce:
- `predicted_truth_score`
- `predicted_utility_score`
- `ranking_position`
- `confidence_interval`

These outputs:
- Must be logged
- Must be signed
- Must be replayable
- **Must never be dereferenced by live code paths**

### B.3 Shadow Violation Detection
A Learning Shadow Violation occurs if:
- A shadow output is read by a non-logging component
- A shadow field is referenced in execution logic
- A shadow value alters state transition

Shadow violations are **P0 incidents**.
