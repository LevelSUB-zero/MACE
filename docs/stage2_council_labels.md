# COUNCIL LABELING RULEBOOK (AUTHORITATIVE)

> **The Core Idea:**
> The Council becomes the ground-truth generator. It labels reality; it does not execute it.

---

## D. COUNCIL = TRUTH LABELER (NOT GOVERNOR)

### D.1 Council Authority Boundary
In Stage-2, the Council **cannot act**.
It may only:
- Label
- Disagree
- Defer
- Conflict

Council outputs are **labels**, not commands.

### D.2 Label Semantics (Formal)
All labels must map to one of these four distinct epistemic axes:

| Label Category | Question Answered | Semantics |
| :--- | :--- | :--- |
| **Truth** | “Is this factually correct?” | Epistemic correctness, independent of usefulness. |
| **Safety** | “Is this dangerous?” | Harm potential to system or user. NOT desirability. |
| **Utility** | “Is this useful?” | Future repair reduction. NOT convenience. |
| **Governance** | “Is this permitted?” | Policy boundary check. NOT endorsement. |

### D.3 Conflict Preservation Doctrine
Conflicts are high-value learning signals.
They must:
- **Remain Visible**: Never hide a split decision.
- **Never be Collapsed**: Do not take a majority vote to "fix" it.
- **Never be Averaged**: Truth is not the mean of error and accuracy.
- **Never be Resolved Automatically**: Only time or amendments resolve conflicts.

A conflict is **information**, not failure.

### D.4 NO_DECISION Semantics
`NO_DECISION` is a valid, affirmative output.
It means: "Insufficient data to judge" or "Outside of domain."
It is distinct from `NULL` or `ERROR`.

### D.5 Label Immutability & Precedence
- **Immutability**: Once a Council session is finalized for a candidate, those labels are **immutable logs**. They cannot be edited, only Amended (see Amendments Rulebook).
- **Precedence**:
  1. **Safety** (Blocking) overrides all.
  2. **Governance** (Blocking) overrides Utility/Truth.
  3. **Truth** ( informative)
  4. **Utility** (informative)

### D.6 Forbidden Council Actions
The Council must NOT:
- Trigger execution of code.
- Write directly to Long-Term Memory (SEM).
- Modify Agent parameters.
- self-optimize its own voting weights.

---

## E. ACCEPTANCE CRITERIA
- Labels are unambiguous.
- Disagreement is preserved, not resolved.
